Kubernetes vs Docker:

People often mistaken Docker and Kubernetes for being the same thing although they are two very different things. Docker is a program that makes and runs containers it was created because when developers would code something on their own computers and push it up for production, problems would arise. Such as different operating systems, or different libraries on the different computers. But containerization negated this problem, by basically having the code separate from the machine's specifications its running on allowing it to run on a different computer.

Kubernetes is not a containerization program rather it is the architect for containers. It help manage the containers rather than actually making them. It shows if there are any errors in the container and alerts the developer and also allows many different containers to work in sync.
